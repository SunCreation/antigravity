    README.md

    README.md — 첫 번째 컨텍스트

    AI도 README를 먼저 읽습니다

  • 무엇을 만드는 프로젝트인가? (한 줄 설명)
  • 어떤 기술 스택을 쓰는가? (Framework, DB, Auth 등)
  • 어떻게 실행/배포하는가? (명령어, 환경변수)
  • 어떤 컨벤션을 따르는가? (코딩 스타일, 네이밍)
  • 주의사항은 무엇인가? (알려진 이슈, 제약사항)
    • 💡
    핵심 원칙
    AI가 README를 읽고 프로젝트의 70%를 이해할 수 있어야 합니다. 나머지 30%는 Skills와 MCP로 보완!

    Documentation

    AI 친화적 README 예시

    # MyApp — 실시간 채팅 서비스
    ## 기술 스택
    - Next.js 15 (App Router) + Firebase + TailwindCSS
    ## 프로젝트 구조
    - src/app/ — 페이지 라우트
    - src/components/ — UI 컴포넌트
    - src/lib/firebase.ts — Firebase 초기화
    ## 컨벤션
    - 컴포넌트: PascalCase, Firestore: snake_case
    - 모든 API 호출은 try/catch로 감싸기
    ## 주의사항
    - Firestore 규칙은 수정 후 반드시 테스트

    AGENTS.md

    AGENTS.md — AI 전용 지시사항

    # AI Agent Instructions
    ## 역할
    너는 이 프로젝트의 풀스택 개발자야.
    ## 코딩 규칙
    - TypeScript strict mode 필수
    - any 타입 절대 사용 금지
    - Firestore 호출은 repository 패턴
    - 에러는 커스텀 AppError 클래스
    ## 금지사항
    - inline style 사용 금지
    - console.log 커밋 금지
    📝
    AGENTS.md의 역할
    README는 "프로젝트 개요"
    AGENTS.md는 "AI에게 주는 작업 지시서"
    ✨
    작성 팁
    ① 구체적일수록 좋다
    ② "하지 마라"보다 "이렇게 해라"
    ③ 예시 코드를 포함하라

    Section 4

    MCP — Model Context Protocol

    AI가 외부 서비스에 직접 접근하게 해주는 프로토콜

    📚
    Agent Skills
    정적 지식 주입 — "어떻게 하는지 알려줘"
    비유: 교과서
    🔌
    MCP
    실시간 데이터 접근 — "직접 확인하고 해줘"
    비유: 인터넷

    MCP를 통해 AI가 Firestore 데이터를 직접 읽고, Auth 사용자를 확인하고, Storage 파일을 관리할 수 있습니다.

    Setup

    Firebase MCP 설정하기

  • Antigravity 설정(Ctrl+,) 열기
  • MCP Servers 섹션 찾기
  • Firebase MCP Server 추가
  • Antigravity 재시작
    • ⏱️
    소요 시간
    설정부터 사용까지 약 2분이면 완료됩니다.

    Configuration

    mcp_config.json 설정 상세

    {
    "mcpServers": {
    "firebase-mcp-server": {
    "command": "npx",
    "args": ["-y", "firebase-tools@latest", "mcp"]
    }
    }
    }
    📝
    설명
    firebase-tools@latest를 통해 항상 최신 MCP 서버를 실행합니다. 실습에서 직접 복사해서 사용할 예정!

    Tools

    Firebase MCP 도구

    💾
    Firestore
    컬렉션 조회, 문서 읽기/쓰기, 쿼리 실행, 인덱스 확인
    🔐
    Authentication
    사용자 목록, 권한 확인, 커스텀 클레임, 세션 관리
    📦
    Storage
    파일 업로드/다운로드, 보안 규칙, Signed URL
    💬
    사용 예시
    "Firestore users 컬렉션에서 최근 가입한 10명을 보여줘" → AI가 직접 Firestore를 조회하여 결과를 반환합니다!

    Decision Guide

    MCP vs Agent Skills: 언제 뭘 쓰나?

    기준Agent SkillsMCP
    역할정적 지식 주입실시간 데이터 접근
    비유교과서, 매뉴얼API, 데이터베이스
    업데이트스킬 설치/업데이트 시항상 최신
    예시"Firestore 컨벤션 따라 코드 작성""현재 Firestore 스키마 확인"
    💡
    모범 사례
    Skills로 "어떻게"를 가르치고, MCP로 "실제 데이터"를 보여주세요. 둘 다 사용할 때 가장 강력합니다.

    Steps

    MCP 설정 3단계 요약

  • mcp_config.json 생성 — 프로젝트 루트에 MCP 서버 설정 파일 생성. Firebase 프로젝트 ID 입력
  • Firebase MCP 패키지 — npx firebase-tools@latest mcp로 바로 실행 가능
  • Antigravity에서 확인 — "Firestore 컬렉션 목록 보여줘" → MCP를 통해 실시간 조회!
    • ⚡
    주의
    Firebase 프로젝트가 배포된 상태여야 MCP가 정상 작동합니다. 실습에서는 미리 준비된 프로젝트를 사용합니다.
