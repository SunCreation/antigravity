Decision

✏️

❌
거부

Reality Check

AI 코드는 프로덕션 레디가 아닙니다

AI가 만든 코드는 초안(Draft)입니다.

80%는 훌륭하지만, 나머지 20%가 문제가 될 수 있습니다.

❌
자주 누락되는 것
에러 처리, 보안 검증, 성능 최적화, 프로젝트 컨벤션
✅
프로덕션에 필요한 것

Checklist

AI 코드 리뷰 체크리스트

기능적 검증

  • 정확성 — 요구사항을 정확히 구현했는가?
  • 에지 케이스 — 경계값 처리가 되어 있는가?
  • 에러 처리 — 예외 상황을 처리했는가?
  • 타입 일관성 — 데이터 타입이 일관적인가?

품질적 검증

  • 가독성 — 다른 개발자가 이해할 수 있는가?
  • 컨벤션 — 프로젝트 코딩 스타일을 따르는가?
  • 중복 코드 — 불필요한 코드가 없는가?
  • 보안 — 입력 검증, 인증/인가가 적절한가?
💡
AI에게도 리뷰 요청
Agent Pane에 "이 코드를 리뷰해줘"라고 요청하면 또 다른 시각에서 분석해줍니다.

Common Mistakes

AI가 자주 하는 실수 TOP 6

👻
허구 API 호출
존재하지 않는 메서드나 함수를 사용. 예: .toogle() ← 오타 그대로 생성

입력 검증 없이 사용자 데이터 처리. 예: SQL 인젝션 방어 없는 쿼리
⚡
성능 무시
O(n²) 알고리즘이나 불필요한 리렌더링
🧩
과도한 추상화
간단한 것을 지나치게 복잡하게 만듦

Refactoring

Agent Pane으로 리팩토링하기

AI가 만든 초안을 AI와 함께 개선할 수 있습니다.

🎯
구체적 지적
"LoginForm에 에러 처리를 추가해줘. 현재 API 실패 시 아무 처리가 없어."
🧩
패턴 적용
"try-catch를 적용하고, 사용자에게 에러 메시지를 보여주는 UI를 추가해줘."
📐
품질 요청
"더 읽기 쉽게 리팩토링해줘. 함수를 작게 나누고, 의미 있는 변수명을 사용해."
✅
핵심
문제를 구체적으로 설명할수록, AI의 리팩토링 품질이 올라갑니다.

Safety

타입 안전성과 에러 처리

❌ AI가 생성한 코드 (위험)

async function fetchUser(id) {
  const res = await fetch(`/api/users/${id}`);
  const data = await res.json();
  return data.name;
}

✅ 개선된 코드 (안전)

async function fetchUser(
  id: string
): Promise<User> {
  if (!id) throw new Error('ID required');
  const res = await fetch(`/api/users/${id}`);
  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  return res.json();
}
⚠️
자주 누락되는 것
API 호출 시 에러 처리, null/undefined 체크, 경계값 검증 — 이 세 가지를 항상 확인하세요.

Conventions

프로젝트 컨벤션 따르기

  • 네이밍 컨벤션 — camelCase? snake_case? 프로젝트 규칙을 AI에게 알려주세요
  • 설정 파일 활용 — .prettierrc, .eslintrc, tsconfig.json
  • CLAUDE.md / .cursorrules — AI에게 코딩 스타일 가이드 제공
  • 예시 코드 제공 — "이 스타일로 만들어줘" + 참고 파일 첨부
  • 리뷰 시 확인 — 네이밍, 포매팅, 구조가 일관적인지

오늘 배운 사이클로 실제 기능을 만들어 봅시다.

📋
과제: 할 일 관리 앱 — "새 할 일 추가" 기능
Step 1: Agent Pane에 기능 기획
Step 2: 코드를 점진적으로 생성
Step 3: 리뷰 & 수정
Step 4: Run and Debug로 테스트

Step 1 · Plan

Step 1: 기능 기획하기 📝

💬
Agent Pane에 입력해보세요
"할 일 관리 앱에 '새 할 일 추가' 기능을 만들어줘. 제목(필수), 설명(선택), 우선순위(높음/중간/낮음), 마감일, 추가 버튼. 빈 제목 시 에러. React + TypeScript + Tailwind."
🎯
구체적으로 작성
필드, 동작, 에러 처리까지 명시
⏱️
시간: 5분
프롬프트를 작성하고 Agent Pane에 입력

Step 2 · Build

Step 2: 코드 생성 & 점진적 확장 ⚡

  • 첫 번째 생성물 검토 — 파일 구조, 임포트, 기본 로직이 맞는지 확인
  • 에러 처리 추가 — "빈 제목 에러 처리를 추가해줘" — 누락된 처리 보완
  • 스타일 조정 — "Tailwind 스타일을 프로젝트에 맞게 조정해줘"
  • 테스트 생성 — "이 컴포넌트의 테스트 코드도 만들어줘"
⏱️
시간: 15분
각 단계에서 생성 → 검토 → 수정 사이클을 돌리세요.

Step 3 · Review

Step 3: 리뷰 & 수정 🔍

리뷰 체크리스트 적용

  • 요구사항 정확히 구현?
  • 에러 처리 누락 없음?
  • 프로젝트 컨벤션 일치?
  • 보안상 문제 없음?

Agent Pane 활용

  • "이 코드를 리뷰해줘"
  • "에러 처리가 빠진 부분 찾아줘"
  • "TypeScript 타입을 더 엄격하게 해줘"
  • 필요한 부분만 선택적 수정

Step 4 · Test

Step 4: 테스트 & (배포) 🚀

  • Run and Debug 메뉴로 실행 — 에러 콘솔 확인
  • 모든 기능 정상 동작 확인 — 직접 클릭, 입력, 제출
  • 문제가 있으면 — 에러 메시지를 Agent Pane에 복사해서 수정 요청
  • 만족하면 — Git 커밋! 시간이 되면 "Publish my app"으로 배포
⏱️
시간: 15분
Run and Debug로 실행해보고, 문제가 있으면 Agent Pane에 에러를 복사해서 수정 요청하세요.
