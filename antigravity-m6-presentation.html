Section 1

프로젝트 시작하기

무엇을 만들지 정하고, 환경을 구축합니다.

코딩은 나중. 먼저 설계부터.

Project

어떤 프로젝트를 만들까?

📋
추천: 할 일 관리 앱
CRUD + 인증 + 실시간 동기화 — 모든 기능을 골고루 경험
💬
대안: 미니 게시판
게시글 작성, 댓글, 사용자 프로필 — Firebase Firestore 활용
ℹ️
왜 Firebase?
Antigravity와 네이티브 통합. 서버리스. 실시간 DB. 배포까지 한 번에.

Requirements

AI와 함께 요구사항 정의하기

Agent에게 프로젝트 아이디어를 이야기하면,
요구사항 명세서를 작성해 줍니다.

💬
프롬프트 예시
"Firebase 기반 할 일 관리 앱을 만들고 싶어. 사용자 인증, CRUD, 카테고리 분류, 마감일 설정 기능이 필요해. 요구사항을 정리해줘."

→ AI가 기능 명세, 사용자 스토리, 기술 스택까지 제안

Setup

Antigravity에서 프로젝트 세팅

  • Firebase Console에서 새 프로젝트 생성
  • Authentication · Firestore 활성화
  • Antigravity에서 프로젝트 폴더 열기
  • firebase init로 Firebase SDK 연동
  • .env 파일에 Firebase 설정값 추가

Skills & MCP

Skills & MCP 설치하기

🔥
Firebase MCP
Firestore CRUD, Auth, Hosting을 Agent에서 직접 제어
🎨
Frontend UI Skill
AI가 UI 컴포넌트를 더 잘 만들도록 지원
⚠️
MCP 설정 팁
mcp_config.json에 Firebase MCP 설정을 추가하세요. 설정 파일은 공유 GitHub에 있습니다.

Structure

AI 친화적 프로젝트 구조

  • CLAUDE.md / AGENTS.md — 프로젝트 설명과 규칙
  • src/ — 명확한 폴더 구조 (components, hooks, lib, types)
  • README.md — AI가 먼저 읽는 파일
  • .env.example — 환경변수 템플릿
  • 명명 규칙 — 일관된 파일/폴더 이름

Section 3

기능 구현 전체 사이클

하나의 기능을 처음부터 끝까지 만들어 봅니다.

정의 → 설계 → 구현 → 테스트 → 리뷰

Define

Step 1: 기능 정의

Agent와 함께 무엇을 만들지 구체화합니다.

💬
프롬프트 예시
"사용자가 할 일을 추가할 수 있는 기능을 만들고 싶어. 제목, 설명, 카테고리, 마감일 필드가 필요해. 사용자 스토리와 엣지 케이스를 정리해줘."

→ AI가 사용자 스토리, 예외 상황, UI 흐름까지 제안

Plan

Step 2: 구현 계획 (Planning Mode)

🚨
Planning Mode 원칙
코드부터 짜지 마세요. 먼저 "어떻게 만들지" 계획을 세우세요.
💬
프롬프트 예시
"할 일 추가 기능을 구현하려고 해. 컴포넌트 구조, 상태 관리, API 호출 흐름을 먼저 계획해줘. 코드는 아직 작성하지 마."

Test

Step 4: 테스트 작성 (Agent-generated)

Agent에게 테스트 코드를 작성해 달라고 요청하세요.

💬
프롬프트 예시
"useTodos 훅에 대한 단위 테스트를 작성해줘. 추가, 삭제, 완료 처리 케이스를 모두 포함해줘."

→ AI가 테스트 파일을 생성하고, 엣지 케이스까지 커버

Debug

Step 5: 디버그 & 수정

  • 에러 메시지를 Agent에게 전달 — 복사해서 그대로 붙여넣기
  • 원인 분석 요청 — "이 에러의 원인이 뭐야?"
  • 수정 제안 받기 — "이 에러를 고쳐줘"
  • 수정 후 재확인 — 테스트를 다시 실행
⚠️
현실적인 조언
AI 코드에도 버그가 있습니다. 당황하지 말고, Agent에게 에러를 보여주세요.

Section 4

테스트 자동 생성

AI 코드는 반드시 테스트로 검증합니다.

테스트는 선택이 아닌 필수

Why Test

AI 코드에 테스트가 중요한 이유

🎯
의도 검증
AI가 내 의도대로 코드를 작성했는지 확인
🔍
엣지 케이스
AI가 놓친 예외 상황을 테스트로 발견
🛡️
회귀 방지
수정해도 기존 기능이 깨지지 않는지 보장
⚠️
핵심 원칙
AI를 믿되, 테스트로 검증하세요. Trust, but verify.

Unit Tests

Agent로 단위 테스트 생성하기

💬
프롬프트 예시
"hooks/useTodos.ts에 대한 단위 테스트를 Vitest로 작성해줘. 추가, 삭제, 토글, 필터링 기능을 모두 테스트하고, Firebase 모킹도 설정해줘."
✅
정상 케이스
할 일 추가 → 목록에 표시
❌
예외 케이스
빈 제목, 네트워크 오류

TDVC

Test-Driven Vibe Coding

📝
테스트 먼저
Agent가 테스트 작성
→
🤖
코드 생성
테스트를 통과하는 코드
→
✅
검증
테스트 통과 확인
→
🔄
리팩토링
안전하게 개선

테스트가 있으니까, AI가 코드를 마음껏 수정해도 안전합니다.

Run Tests

Antigravity에서 테스트 실행하기

  • 터미널에서 npm run test 또는 npx vitest 실행
  • 테스트 결과를 Agent Pane에 복사
  • 실패한 테스트에 대해 "원인 분석해줘" 요청
  • Agent가 수정 코드를 제안 → 수락 → 재실행
✅
워크플로우
테스트 실패 → Agent 분석 → 수정 → 재실행 → 통과. 이 사이클이 바이브 코딩의 핵심입니다.
