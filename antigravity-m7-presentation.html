Setup

Firebase 배포 설정하기

  • Firebase Console에서 프로젝트 생성
  • Firebase CLI 설치: npm install -g firebase-tools
  • 로그인: firebase login
  • 초기화: firebase init hosting
  • 빌드: npm run build
⚠️
주의
firebase init 시 public 디렉토리를 dist 또는 build로 설정하세요.

Lessons

Lessons Learned

🎯
명확한 요구사항
좋은 프롬프트 = 좋은 결과. AI에게도 구체적인 목표가 필요합니다.
🔄
반복 개선
한 번에 완벽한 코드는 없습니다. Agent와의 대화를 통해 점진적으로 개선하세요.
🧪
테스트의 가치
작은 테스트라도 작성하면, Agent가 수정해도 안심할 수 있습니다.
🤝
협업의 힘
AI는 도구이고, 최종 결정은 항상 개발자의 몫입니다.

Principles

세 가지 핵심 원칙

🧠
Planning Mode
코딩하기 전에 먼저 생각하기. 무엇을 만들지, 어떻게 만들지 명확히 정의
📐
Context Engineering
AI에게 프로젝트 컨텍스트를 제공. 좋은 질문이 좋은 답을 만든다
🧱
Incremental Build
작은 단위로 만들고 테스트. 한 번에 완벽한 코드는 없다

이 세 가지는 Antigravity를 넘어 모든 AI 코딩 도구에 적용되는 원칙입니다.

Trust & Verify

AI를 신뢰할 때, 검증할 때

상황접근이유
보일러플레이트 코드AI 신뢰 ✓패턴이 명확하고 위험도 낮음
스타일링 / UIAI 신뢰 ✓시각적으로 바로 확인 가능
비즈니스 로직반드시 검증도메인 지식이 필요
보안 / 인증반드시 검증취약점이 치명적
성능 최적화반드시 검증측정 없이는 개선을 알 수 없음

Pitfalls

자주하는 실수와 해결책

실수해결책
AI 결과를 무조건 수용생성 후 반드시 읽고 이해하기
컨텍스트 없이 질문관련 파일을 먼저 열어두기
한 번에 전체 앱 요청기능 단위로 나누어 요청
테스트 코드 작성 안 함Agent에게 테스트도 함께 요청
배포가 어려울 것이라고 생각Firebase로 명령 하나면 충분
